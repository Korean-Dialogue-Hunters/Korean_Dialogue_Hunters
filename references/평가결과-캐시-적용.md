# 평가 결과 캐시 적용 — 히스토리에서 결과 재조회 시 중복 API 호출 방지

## 문제

대화 결과를 이미 생성했는데, `/history`에서 해당 기록 카드를 다시 클릭하면 **백엔드가 새로운 평가를 다시 생성**한다.

### 원인 분석

1. `app/history/page.tsx:46-49` — `handleCardClick`이 **sessionId만** sessionStorage에 넣고 `/result`로 이동
2. `app/result/page.tsx:70-101` — `/result` 페이지는 **무조건** `evaluateSession(sessionId)` API를 호출
3. 이미 평가된 결과인지 확인하는 로직이 없음

```
현재 흐름:
history 카드 클릭
  → sessionStorage.setItem("sessionId", ...)
  → router.push("/result")
  → evaluateSession(sessionId)   ← 매번 새로 호출!
  → POST /sessions/{id}/evaluation
```

## 해결 방향

localStorage에 전체 `EvaluationResponse`를 sessionId 기준으로 캐시하고, `/result` 페이지에서 캐시가 있으면 API 호출을 건너뛴다.

```
목표 흐름:
[최초 평가] /result 진입
  → localStorage에 캐시 확인 → 없음
  → evaluateSession() API 호출
  → 결과를 localStorage 캐시에 저장 + 기존 로직 수행

[재조회] history 카드 클릭 → /result 진입
  → localStorage에 캐시 확인 → 있음
  → 캐시된 데이터 사용 (API 호출 안 함)
```

## 수정 대상 파일 (3개)

### 1. `lib/historyStorage.ts` — 평가 캐시 CRUD 함수 추가

**변경 내용**: 기존 `dialogueHistory`와 별도로 `evaluationCache`라는 localStorage 키를 추가한다. sessionId를 키로 전체 `EvaluationResponse` 객체를 저장/조회/삭제한다.

```ts
// 파일 상단에 상수 추가
const EVAL_CACHE_KEY = "evaluationCache";

// 아래 함수들을 파일 하단에 추가 (export)

/** 평가 결과 캐시 전체 조회 (내부 헬퍼) */
function getEvaluationCacheAll(): Record<string, unknown> {
  if (typeof window === "undefined") return {};
  try {
    const raw = localStorage.getItem(EVAL_CACHE_KEY);
    return raw ? JSON.parse(raw) : {};
  } catch {
    return {};
  }
}

/** 평가 결과 캐시 저장 */
export function saveEvaluationCache(sessionId: string, data: unknown): void {
  if (typeof window === "undefined") return;
  try {
    const cache = getEvaluationCacheAll();
    cache[sessionId] = data;
    localStorage.setItem(EVAL_CACHE_KEY, JSON.stringify(cache));
  } catch {
    // localStorage 용량 초과 등 무시
  }
}

/** 평가 결과 캐시 조회 — 없으면 null */
export function getEvaluationCache(sessionId: string): unknown | null {
  const cache = getEvaluationCacheAll();
  return cache[sessionId] ?? null;
}

/** 평가 결과 캐시 삭제 */
export function removeEvaluationCache(sessionId: string): void {
  if (typeof window === "undefined") return;
  const cache = getEvaluationCacheAll();
  delete cache[sessionId];
  localStorage.setItem(EVAL_CACHE_KEY, JSON.stringify(cache));
}
```

### 2. `app/result/page.tsx` — 캐시 우선 조회, 없을 때만 API 호출

**import 변경**:
```ts
// 기존
import { addHistory } from "@/lib/historyStorage";
// 변경
import { addHistory, getEvaluationCache, saveEvaluationCache } from "@/lib/historyStorage";
```

**useEffect 내부 로직 변경** (70~101번 줄):

기존 코드:
```ts
useEffect(() => {
  const sessionId = sessionStorage.getItem("sessionId");
  if (!sessionId) {
    router.replace("/");
    return;
  }

  evaluateSession(sessionId)
    .then((res) => {
      setEvalData(res);
      setScores(extractScores(res));
      sessionStorage.setItem("evaluationData", JSON.stringify(res));
      addHistory({ ... });
    })
    .catch((e) => {
      setError(e instanceof Error ? e.message : "평가 요청에 실패했습니다");
    })
    .finally(() => setLoading(false));
}, [router]);
```

변경 코드:
```ts
useEffect(() => {
  const sessionId = sessionStorage.getItem("sessionId");
  if (!sessionId) {
    router.replace("/");
    return;
  }

  /* 1) localStorage 캐시 확인 */
  const cached = getEvaluationCache(sessionId) as EvaluationResponse | null;
  if (cached) {
    setEvalData(cached);
    setScores(extractScores(cached));
    sessionStorage.setItem("evaluationData", JSON.stringify(cached));
    setLoading(false);
    return;
  }

  /* 2) 캐시 없음 → API 호출 (최초 평가) */
  evaluateSession(sessionId)
    .then((res) => {
      setEvalData(res);
      setScores(extractScores(res));
      sessionStorage.setItem("evaluationData", JSON.stringify(res));
      /* 캐시 저장 */
      saveEvaluationCache(sessionId, res);
      /* 대화 기록에 저장 */
      addHistory({
        sessionId: res.sessionId,
        scenarioTitle: res.scenarioTitle,
        location: res.location,
        scene: res.scene ?? "",
        totalScore10: res.totalScore10,
        grade: res.grade,
        feedback: res.feedback,
        llmSummary: res.llmSummary,
        turnCount: res.conversationLog.filter((e) => e.role === "user").length,
        createdAt: new Date().toISOString(),
      });
    })
    .catch((e) => {
      setError(e instanceof Error ? e.message : "평가 요청에 실패했습니다");
    })
    .finally(() => setLoading(false));
}, [router]);
```

핵심: `getEvaluationCache(sessionId)`로 먼저 확인 → 있으면 바로 사용, 없으면 기존 API 호출 + `saveEvaluationCache()`로 캐시 저장.

## 검증 방법

1. 새 대화 완료 → `/result` 진입 → API 호출됨 (정상)
2. 브라우저 DevTools > Application > localStorage에서 `evaluationCache` 키에 데이터 저장 확인
3. 홈으로 이동 → `/history` → 방금 완료한 카드 클릭 → `/result` 진입
4. **Network 탭에서 `/evaluation` 요청이 발생하지 않으면 성공**
5. 표시되는 점수/등급/요약이 최초 평가 때와 동일한지 확인

## 주의사항

- `EvaluationResponse` 객체는 꽤 크다 (conversationLog 포함). 대화가 많이 쌓이면 localStorage 용량 한계(~5MB)에 도달할 수 있다. 추후 오래된 캐시를 자동 정리하는 로직이 필요할 수 있음.
- `removeEvaluationCache()`는 지금 당장 사용하지 않지만, 히스토리 삭제 기능 추가 시 `removeHistory()`와 함께 호출해야 한다.
